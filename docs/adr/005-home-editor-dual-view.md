# ADR-005: 双页面架构（首页 + 编辑器）

**日期:** 2026-07-15
**状态:** Accepted

## 背景

当前 Recordly 使用 QStackedWidget 在"编辑器"和"项目文件"两个页面间通过菜单互斥切换。启动默认进入编辑器，所有控件禁用，体验不佳。录制完成后停留在编辑器，用户感知不到项目已自动创建。

## 决策

采用**首页 → 编辑器**的双页面架构，替代现有的"编辑器 ↔ 项目文件"模式：

1. **首页（HomePage）** — 启动默认页，包含操作按钮（开始录制/打开项目）+ 项目画廊
2. **编辑器（EditorInterface）** — 录制/编辑页面，通过工具栏操作

页面切换方向：
- 首页 → 编辑器：录制完成自动切 / 点击项目卡片
- 编辑器 → 首页：菜单"返回首页"

## 理由

- **符合用户心智模型**：先选择/创建项目，再进入编辑
- **减少认知负担**：启动后直接看到可操作入口，而非灰色禁用控件
- **单向流动**：首页 → 编辑器 → 返回首页，比双向 toggle 更清晰
- **复用现有组件**：ProjectGallery、ProjectCard 无需修改

## 备选方案

1. **保留三页面（首页 + 编辑器 + 项目列表）** — 增加复杂度，项目列表可合入首页
2. **弹窗模式** — 录制/项目选择用对话框，不符合桌面应用惯例

## 影响

- 新增 `ui/home_page.py`
- `MainWindow._stacked_widget` 从 2 页（编辑/项目）改为 2 页（首页/编辑器）
- 菜单栏移除"视图"菜单
- 工具栏在首页隐藏
