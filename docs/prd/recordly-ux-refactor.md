# PRD: Recordly 交互与页面架构重构

**日期:** 2026-07-15
**状态:** Draft

## 1. 概述

### 1.1 问题陈述
当前 Recordly 的交互逻辑存在以下问题：
- 启动默认进入编辑器页面，所有控件禁用，用户体验差
- 视图切换逻辑反直觉（编辑器 ↔ 项目文件通过菜单互斥切换）
- 录制完成后自动创建了项目但停留在编辑器，用户感知不到项目存在
- 编辑器内没有返回首页的入口
- 菜单栏/QToolBar 职责不清，导航项和功能按钮混在一起

### 1.2 目标用户
使用 Recordly 进行屏幕录制和视频编辑的用户

### 1.3 成功指标
- 启动后 1 秒内展示可交互的项目选择页面
- 录制完成后自动切换到编辑器并可立即回放
- 从首页到编辑器、编辑器回首页的操作步骤 ≤ 2 步

## 2. 功能需求

### 2.1 核心功能（MVP）
- [ ] F1: 首页（HomePage） — 启动默认展示，包含开始录制/打开项目按钮 + 项目画廊
- [ ] F2: 录制确认弹窗 — 点击"开始录制"后弹出确认对话框，确认后窗口最小化开始录屏
- [ ] F3: 录制完成自动切换 — 停止录制 → 自动创建项目 → 自动切换到编辑器
- [ ] F4: 按页面显示/隐藏工具栏 — 首页无工具栏，编辑器工具栏含所有功能按钮
- [ ] F5: 返回首页 — 编辑器菜单"返回首页"切换到首页并刷新项目列表
- [ ] F6: 点击项目进入编辑器 — 项目画廊中点击卡片加载并进入编辑器

### 2.2 扩展功能（后续迭代）
- [ ] E1: 首页最近项目与全部项目分区域展示
- [ ] E2: 打开项目加载视频帧到编辑器
- [ ] E3: 编辑器内"保存"按钮手动保存项目状态

### 2.3 非功能需求
- 性能：页面切换 < 200ms
- 可用性：首页所有可交互元素有明确的视觉反馈

## 3. 用户故事

### US-1: 首次使用 — 从首页开始录制
**作为** 新用户
**我想要** 打开软件后看到一个清晰的操作入口
**以便** 快速开始录制而不需要理解复杂的编辑器

**验收标准：**
- [ ] 启动后显示首页（HomePage），含"开始录制"和"打开项目"按钮
- [ ] 点击"开始录制"弹出确认对话框
- [ ] 确认后窗口最小化，开始屏幕录制
- [ ] 停止录制后自动创建项目，切换到编辑器页面

### US-2: 浏览已有项目
**作为** 老用户
**我想要** 在首页看到所有项目并点击进入
**以便** 继续之前的工作

**验收标准：**
- [ ] 首页下方展示所有项目卡片（复用 ProjectGallery）
- [ ] 点击卡片加载项目并切换到编辑器
- [ ] 返回首页时项目列表已刷新

### US-3: 录制后立即编辑
**作为** 用户
**我想要** 录制完成后直接进入编辑页面
**以便** 无缝衔接录制和编辑流程

**验收标准：**
- [ ] 停止录制后自动切换到编辑器
- [ ] 工具栏按钮可用（播放、导出等）
- [ ] 时间线已填充录制内容

### US-4: 从编辑器返回首页
**作为** 用户
**我想要** 从编辑器返回首页查看项目列表
**以便** 切换项目或开始新录制

**验收标准：**
- [ ] 编辑器菜单"文件 → 返回首页"可切换到首页
- [ ] 返回后项目列表已刷新

## 4. 约束与假设

### 4.1 技术约束
- 技术栈：PyQt5，不引入新 UI 框架
- 复用现有 ProjectGallery / ProjectCard 组件

### 4.2 业务约束
- 本次不实现视频帧解码加载（打开项目后预览区仍为空，后续迭代）

### 4.3 假设
- 录制功能（Recorder/Compositor）工作正常，本次不修改录制核心逻辑
- 项目持久化（ProjectManager.create_project / save）已正常工作

- 视频帧解码到 Compositor（打开项目后预览区加载视频帧）
- 编辑器 UI 组件内部重构（PreviewWidget / TimelineWidget）
- 导出流程修改
- 项目导入/导出的文件管理器交互
