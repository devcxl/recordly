# PRD: Recordly 核心稳定性与架构治理

**日期：** 2026-07-16  
**状态：** Approved  
**来源：** `docs/review/core-architecture-interaction-review-2026-07-16.md`

## 1. 概述

### 1.1 问题陈述

Recordly 当前录制、项目持久化、重新打开、播放和导出链路存在多项发布阻断问题：已有项目导出崩溃、录音未持久化、音频与视频时间基准错误、录制失败无法可靠恢复、导出线程可能挂起，以及测试基线失败。与此同时，`MainWindow` 承担过多职责并跨模块访问私有状态，使相同业务动作出现多套不一致流程。

### 1.2 目标用户

使用 Recordly 本地桌面应用录制教程、产品演示或问题复现视频的个人用户，包括开发者和内容创作者。

### 1.3 业务价值

- 防止录制数据丢失或已有项目被意外破坏。
- 保证项目关闭并重新打开后仍可正确播放和导出。
- 保证 MP4/GIF 导出可终止、音画同步且不残留资源。
- 降低后续功能开发因状态耦合造成的回归风险。

### 1.4 成功指标

1. 全量测试 `0 failed`，不新增用于绕过缺陷的 `skip/xfail`。
2. 审查报告中的核心交互流程全部通过自动化回归。
3. MP4/GIF 导出时长误差不超过 1 帧。
4. 音频时间线定位误差不超过 1 个音频采样块或 1 帧视频时长。
5. 取消或失败导出后无 FFmpeg 残留进程、临时 WAV 或不完整目标文件。
6. 项目保存异常时，原 `project.json` 保持可读取。
7. `MainWindow` 不超过 800 行，且不直接访问跨模块私有字段。
8. 最终代码审查不存在 Critical/High 问题。

## 2. 功能需求

### 2.1 P0：发布阻断修复

- [ ] F1：恢复测试基线，修复当前 11 个失败测试及其暴露的生产问题。
- [ ] F2：从首页项目卡片或 `project.json` 文件打开项目后，可正常播放和导出。
- [ ] F3：项目输入在 UI/Session 边界统一规范化为项目目录。
- [ ] F4：打开无效或不兼容项目失败时，当前编辑状态保持不变并显示明确错误。
- [ ] F5：每次首页或托盘录制都自动创建独立项目，不复用当前项目路径。
- [ ] F6：录制启动、停止及失败恢复使用同一状态机。
- [ ] F7：麦克风与系统音频分别持久化为项目内 WAV 文件，重新打开后可恢复混音播放和导出。
- [ ] F8：音频源裁剪区间与时间线放置时间严格分离，移动、裁剪和变速后保持音画同步。
- [ ] F9：已有项目始终按项目原始 FPS 导出，不受当前默认录制 FPS 影响。
- [ ] F10：ExportWorker 在成功、失败、取消和异常路径恰好产生一次完成结果，UI 线程正常退出。
- [ ] F11：MP4/GIF 持续消费 FFmpeg stderr，长视频导出不得因管道填满而死锁。
- [ ] F12：取消导出时终止并等待 FFmpeg，删除临时音频和未完成目标文件，UI 显示“导出已取消”。

### 2.2 P1：安全与架构治理

- [ ] F13：使用 `NamedTemporaryFile` 或 `mkstemp` 替代 `tempfile.mktemp`。
- [ ] F14：项目元数据通过同目录临时文件和 `os.replace` 原子保存。
- [ ] F15：统一导出进程、管道、reader thread 和临时文件生命周期清理。
- [ ] F16：提取 `ProjectSession`，拥有当前项目目录、Project 模型和媒体资源路径契约。
- [ ] F17：提取 `RecordingController`，拥有录制状态、项目创建和失败恢复。
- [ ] F18：提取 `ExportController`，拥有 QThread、进度、取消和完成状态。
- [ ] F19：`MainWindow` 缩减到 800 行以内，不再访问 recorder、compositor、timeline、playback 的跨模块私有字段。
- [ ] F20：统一光标插值实现并缓存时间索引，避免逐帧重复线性扫描。

### 2.3 P2：质量收尾

- [ ] F21：使用 Python `logging`，默认仅输出 Warning/Error；`RECORDLY_DEBUG=1` 开启 Debug 并输出到 stderr。
- [ ] F22：移除无条件帧级/FPS stderr 输出和动态 `__import__` 日志调用。
- [ ] F23：将 `main.py` 大型 QSS 移至 `resources/style.qss`。
- [ ] F24：统一 `FrameStyle.bg_color` 的运行时类型、序列化格式与测试契约。
- [ ] F25：新增或修改的函数原则上不超过 50 行。

### 2.4 非功能需求

- **可靠性：** 所有失败路径必须恢复 UI 到可继续操作状态，不能破坏已有项目。
- **数据安全：** 保存操作原子化；临时文件安全创建；失败时优先保留可恢复录制。
- **性能：** 光标/相机插值使用二分索引；不得引入逐帧 O(n) 扫描。
- **可维护性：** UI、项目会话、录制和导出状态所有权清晰。
- **兼容性：** Linux 完整验收；Windows 不得引入平台相关回归。
- **可观测性：** 默认无调试噪声；Debug 模式可诊断但不得输出完整用户内容或音频数据。

## 3. 用户故事

### US-1：可靠录制与恢复

**作为** Recordly 用户  
**我想要** 从首页或托盘开始录制时总是创建独立项目  
**以便** 新录制不会丢失或破坏已有项目。

**验收标准：**

- [ ] 每次录制均创建唯一项目目录。
- [ ] 启动失败且没有帧时自动删除占位项目。
- [ ] 停止失败但帧可读取时保留恢复项目并恢复窗口。
- [ ] 数据不可读取时删除损坏项目并明确提示。

### US-2：项目持久化与重新打开

**作为** Recordly 用户  
**我想要** 保存并重新打开项目后继续播放和导出  
**以便** 工作不受应用重启影响。

**验收标准：**

- [ ] `project.json` 文件选择和项目卡片打开路径都成功。
- [ ] 视频帧、光标、点击、时间线、麦克风和系统音频完整恢复。
- [ ] 打开失败不修改当前会话。
- [ ] 保存中断不损坏原项目元数据。

### US-3：正确导出

**作为** Recordly 用户  
**我想要** 导出的 MP4/GIF 与编辑时间线一致  
**以便** 获得时长正确、音画同步的视频。

**验收标准：**

- [ ] 已打开项目导出不访问空运行时录制数据。
- [ ] 导出使用项目 FPS，时长误差不超过 1 帧。
- [ ] 音频裁剪使用源区间，延迟使用时间线位置。
- [ ] FFmpeg 启动失败、BrokenPipe 或渲染异常均返回失败结果并关闭线程。
- [ ] 长 GIF 导出不因 stderr 阻塞。
- [ ] 取消后无残留进程、临时文件和不完整目标文件。

### US-4：可维护架构

**作为** Recordly 维护者  
**我想要** 项目、录制和导出状态由独立对象管理  
**以便** 后续修改不会再次产生多套不一致流程。

**验收标准：**

- [ ] `ProjectSession`、`RecordingController`、`ExportController` 职责独立。
- [ ] `MainWindow ≤ 800` 行。
- [ ] `MainWindow` 不跨模块读取或修改私有字段。
- [ ] 核心交互由集成测试覆盖。

## 4. 约束与假设

### 4.1 技术约束

- 保持 Python、PyQt5、FFmpeg、NumPy、Pillow、OpenCV 现有技术栈。
- 不引入新的运行时依赖，除非技术方案证明不可替代。
- 继续使用 pytest；测试替身必须模拟明确接口，不能依赖全局 MagicMock 隐式吞掉行为。
- Linux 为完整验收平台；Windows 保持现有兼容性。
- 导出保持项目原始 FPS，不实现帧率重采样。

### 4.2 业务约束

- 所有审查问题均纳入本轮，明确列入 Out-of-Scope 的项目除外。
- 按 P0 → P1 → P2 分批交付，每批均须独立测试和审查。
- 不以大规模一次性重写替代渐进式重构。

### 4.3 假设

- 当前项目格式是本轮唯一保证正常打开的格式。
- 麦克风和系统音频按现有规则混音；本轮不新增独立调音 UI。
- FFmpeg 已作为系统依赖安装；缺失时应用必须优雅失败。

## 5. 测试与验收

### 5.1 必须自动化的回归场景

- [ ] 首页录制启动失败后恢复窗口、状态和按钮并清理占位项目。
- [ ] 屏幕采集在停止时失败后恢复窗口并正确处理项目目录。
- [ ] 托盘录制创建新项目且不修改当前项目。
- [ ] 选择 `project.json` 能正确打开项目。
- [ ] 打开损坏或不兼容项目失败后当前状态不变。
- [ ] 项目卡片打开后可正常导出。
- [ ] 双音轨保存并重新打开后可播放和导出。
- [ ] 不同项目 FPS 与应用默认 FPS 组合下导出时长正确。
- [ ] 音频移动、裁剪和视频变速后源区间与时间线位置正确。
- [ ] FFmpeg 启动失败后 worker、thread 和进度框全部结束。
- [ ] GIF 大量 stderr 输出时不死锁。
- [ ] 取消导出后无资源残留。
- [ ] 项目保存异常时原 JSON 保持可读。

### 5.2 完成定义

- [ ] 全量测试 0 failed。
- [ ] Linux 核心流程实测通过。
- [ ] Windows 兼容性检查无新增回归。
- [ ] 最终审查不存在 Critical/High。
- [ ] 所有 Parent/Sub Issues 验收标准完成。
- [ ] Felix 完成最终产品与技术验收。

## 6. 不在范围内

- 历史项目自动迁移。
- macOS 系统音频录制支持。
- FPS 转换或帧重采样。
- 麦克风和系统音轨独立调音 UI。
- 日志文件、日志轮转及远程错误上报。
- 新增编辑功能、云同步、账号系统。
- Release、安装包发布和应用商店分发。

详见 `docs/dev/out-of-scope.md`。
