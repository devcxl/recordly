# PRD: 播放头点击行为修正

**日期:** 2026-07-19
**状态:** Draft

## 1. 概述

### 1.1 问题陈述

`TimelineWidget.mousePressEvent` 当前的执行顺序是：**先更新播放头位置**，再判断鼠标下方是否有 Clip。这导致两个问题：

1. **播放头干扰编辑**：点击片段（尤其是小片段）时，播放头跳到点击位置，视觉上挡住片段，干扰后续操作
2. **无法实现"双击播放"**：当前双击事件仅处理缩放轨道，视频/音频轨道双击无任何行为

### 1.2 目标用户

所有使用 Recordly 时间线编辑的用户。

### 1.3 成功指标

- 单击片段 → 选中片段，播放头不移动
- 单击空白区域 → 播放头跳转到该位置（seek）
- 双击空白区域 → 播放头跳转 + 从该位置开始播放

## 2. 功能需求

### 2.1 F1：点击片段时不移播放头

**当前行为**（`timeline.py:202-245`）：
```python
def mousePressEvent(self, event):
    # 1. 先移动播放头
    self._playhead_s = self._x_to_time(int(pos.x()))
    self.playhead_changed.emit(self._playhead_s)
    # 2. 再判断是否点到 clip
    ti, ci = self._hit_test(pos)
    if ti >= 0 and ci >= 0:
        self._drag_state = "move"   # 覆盖为 move，但播放头已移动
```

**期望行为**：
- 先 `_hit_test(pos)` 判断鼠标下方是否有 clip
- 有 clip → 选中该片段，设置 `_drag_state = "move"`，**不更新播放头**
- 无 clip → 更新播放头位置（现有逻辑），设置 `_drag_state = "playhead"`

**边界条件：**
- 点击 clip 边缘（resize_left/resize_right 拖拽区）→ 同样不移播放头
- 点击缩放轨道 clip → 同样不移播放头
- 点击标尺区域（`pos.y() < RULER_HEIGHT`）→ 始终移动播放头（不受 clip 影响）

### 2.2 F2：单击空白区域 = Seek

- **行为**：点击无 clip 的轨道区域，播放头跳转到该时间位置
- **与现有行为一致**，仅需确保 F1 修正后不破坏此逻辑

### 2.3 F3：双击空白区域 = Seek + Play

- **新增信号**：`TimelineWidget` 新增 `playhead_seek_play = pyqtSignal(float)`
- **触发条件**：
  - 双击位置在标尺下方（`pos.y() >= RULER_HEIGHT`）
  - 双击位置不在任何 clip 上方（`_hit_test(pos)` 无结果）
  - 双击位置不在缩放轨道（zoom track 双击有独立行为，不覆盖）
- **行为**：
  1. 播放头移动到双击位置
  2. 发射 `playhead_seek_play` 信号，携带时间位置
  3. `MainWindow` 连接该信号 → seek + 开始播放
- **焦点排除**：同 Space 快捷键规则，输入框/弹窗获得焦点时不处理

## 3. 用户故事

### US-1: 点击片段不干扰编辑
**作为** 视频编辑用户
**我想要** 点击视频片段时播放头不跟着跳过来
**以便** 编辑小片段时不被播放头遮挡

**验收标准：**
- [ ] 单击视频/音频/缩放片段，播放头位置不变
- [ ] 单击片段边缘（拖拽区域），播放头位置不变
- [ ] 片段仍可正常选中、拖拽、缩放
- [ ] 现有的片段选中逻辑（高亮、属性面板更新）不受影响

### US-2: 单击跳转播放头
**作为** 视频编辑用户
**我想要** 单击时间线空白区域时播放头跳转到该位置
**以便** 快速定位到特定时间点

**验收标准：**
- [ ] 单击无 clip 的轨道区域，播放头跳转到该位置
- [ ] 单击标尺区域，播放头跳转到该位置
- [ ] 播放头跳转后预览画面更新到对应帧

### US-3: 双击跳转并播放
**作为** 视频编辑用户
**我想要** 双击时间线空白区域时播放头跳转并开始播放
**以便** 快速从指定位置预览

**验收标准：**
- [ ] 双击无 clip 区域，播放头跳转 + 自动开始播放
- [ ] 播放按钮状态同步为"暂停"图标
- [ ] 双击缩放轨道区域 → 不触发播放（保持现有缩放块创建行为）
- [ ] 双击在输入框/弹窗焦点时不触发

## 4. Out of Scope

| 排除项 | 原因 |
|--------|------|
| 播放头视觉样式调整（透明度/颜色） | 行为修正已解决核心痛点 |
| 双击片段的行为变更 | 保持现有选中+拖拽语义 |
| 播放头吸附 | 已在 timeline-interaction-enhancements 中排除 |
| JKL 穿梭播放 | 属于播放控制范畴，应单独立项 |
| 三击行为 | 无实际需求 |

## 5. 技术约束

- 语言：Python 3.11+
- UI 框架：PyQt5
- `mousePressEvent` 调整执行顺序：先 `_hit_test`，再决定是否移动播放头
- `mouseDoubleClickEvent` 新增空白区域双击处理，与缩放轨道双击互斥
- 新增 `playhead_seek_play` 信号，`MainWindow` 中连接并实现 seek+play
- 焦点排除复用现有 `_on_space_shortcut()` 的 `activeModalWidget`/`activePopupWidget`/`focusWidget` 检查模式
