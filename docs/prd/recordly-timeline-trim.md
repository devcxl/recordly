# PRD: 时间线裁剪功能完善

**日期:** 2026-07-17
**状态:** Draft

## 1. 概述

### 1.1 问题陈述

用户需要对录制视频/音频进行长度裁剪：去掉开头、结尾或中间不需要的片段。

当前 Timeline 已内置 Clip 边缘拖拽（`resize_left`/`resize_right`）和拆分（`SplitClipCommand`）能力，但存在以下缺陷：

1. **边缘拖拽不更新 source 范围**：`ui/timeline.py:242-250` 的 `resize_left`/`resize_right` 只修改 `clip.start`/`clip.end`，不更新 `clip.source_start`/`clip.source_end`。对于有显式 `source_end` 的 clip（如拆分后的 clip），音频导出时 `exporter.py:616-618` 仍使用旧的 source 范围，导致音画长度不匹配。
2. **发现性差**：边缘拖拽无视觉提示（无 hover 光标变化、无拖拽手柄），用户不知道 clip 边缘可拖拽。
3. **中间裁剪需两步**：裁掉中间一段需要 S 键拆分 → Delete 删除，操作繁琐。
4. **无快速裁剪入口**：缺少"以播放头为界一键裁剪"的快捷操作。

### 1.2 目标用户

录制屏幕后需要去除多余片段（开头准备动作、结尾等待、中间误操作段落）的所有 Recordly 用户。

### 1.3 成功指标

- 拖拽 clip 边缘后，视频预览和导出音频长度一致
- 拆分过的 clip 再做边缘拖拽，导出结果正确
- 鼠标悬停在 clip 边缘时光标变为水平调整箭头
- 播放头定位后可按快捷键一键裁掉之前/之后的内容

## 2. 功能需求

### 2.1 核心功能（MVP）

- [ ] **F1**：修复 `resize_left`/`resize_right` — 边缘拖拽时同步更新 `source_start`/`source_end`，确保音频导出与视觉裁剪一致
- [ ] **F2**：Clip 边缘 hover 光标变化 — 鼠标靠近左/右边缘时显示 `SizeHorCursor`
- [ ] **F3**：播放头定位后一键裁剪 — 快捷键裁掉播放头之前或之后的内容，复用现有 SplitClipCommand + DeleteClipCommand

### 2.2 扩展功能（后续迭代）

- [ ] E1：视觉拖拽手柄（clip 边缘显示小块竖线提示可拖拽）
- [ ] E2：Ripple Delete（删除 clip 后自动前移后续 clip）
- [ ] E3：裁剪预览（拖拽时实时更新预览画面）

### 2.3 非功能需求

- 裁剪操作必须可撤销（Ctrl+Z），复用现有 `UndoCommand` 体系
- 裁剪后 project.json 的 clip 数据中 source_start/source_end 与 start/end 保持一致

## 3. 用户故事

### US-1: 拖拽边缘裁剪并正确导出
**作为** 用户
**我想要** 拖拽 clip 的左/右边缘来修剪开头/结尾，并且导出后音频与视频长度一致
**以便** 快速去掉录制开头或结尾不需要的部分

**验收标准：**
- [ ] 拖拽 clip 左边缘后，`source_start` 随 `start` 同步增加
- [ ] 拖拽 clip 右边缘后，`source_end`（如非 None）随 `end` 同步更新
- [ ] 拆分后的 clip 再做边缘拖拽，导出时音频长度与视频一致
- [ ] 操作可撤销/重做

### US-2: 以播放头为界快速裁剪
**作为** 用户
**我想要** 将播放头定位到某处后一键裁掉当前位置之前（或之后）的所有内容
**以便** 不用拆分+删除两步就能完成裁剪

**验收标准：**
- [ ] "裁剪开头"操作：删除 playhead 之前的所有 clip 内容（clip 左边缘移到 playhead 处）
- [ ] "裁剪结尾"操作：删除 playhead 之后的所有 clip 内容（clip 右边缘移到 playhead 处）
- [ ] 操作可撤销/重做
- [ ] 快捷键（如 I 键裁入点、O 键裁出点）

### US-3: Clip 边缘可拖拽提示
**作为** 用户
**我想要** 鼠标靠近 clip 边缘时光标变化
**以便** 知道剪辑边缘可以拖拽

**验收标准：**
- [ ] 鼠标在 clip 左/右边缘 8px 内时，光标变为水平调整箭头
- [ ] 离开边缘区域后光标恢复正常

## 4. Out of Scope

| 排除项 | 原因 |
|--------|------|
| 视觉拖拽手柄（E1） | 先用光标提示检验可用性，后续迭代再加 |
| Ripple Delete（E2） | 增加复杂度，先保证基础裁剪正确 |
| 多轨裁剪联动 | 当前仅视频+音频双轨，无多轨场景 |
| 裁剪预览（E3） | 拖拽时预览帧刷新已有性能问题，需单独优化 |
| JKL 精确定位 | 属于播放控制范畴，非裁剪专属 |

## 5. 技术约束

- 语言：Python 3.11+
- UI 框架：PyQt5
- 裁剪命令必须继承 `UndoCommand`，纳入现有 undo/redo 栈
- `source_start`/`source_end` 的更新逻辑需与 `SplitClipCommand` 保持一致（按 speed 比例换算）
