# PRD: 时间线编辑器交互增强

**日期:** 2026-07-19
**状态:** Draft

## 1. 概述

### 1.1 问题陈述

Recordly 时间线编辑器缺少三方面效率提升能力：

1. **键盘效率低**：播放/暂停必须用鼠标点击工具栏按钮，切割片段必须用 S 键且需先选中片段
2. **片段对齐靠手动**：两段视频片段贴近后存在微小间隙或重叠，无法自动消除
3. **缩放块入口隐蔽**：手动添加缩放块仅靠双击缩放轨道空白区域，无右键菜单、无工具栏按钮

### 1.2 目标用户

使用 Recordly 编辑屏幕录制的所有用户。

### 1.3 成功指标

- Space 键可切换播放/暂停，无需鼠标操作
- X 键在播放头处即时切割视频片段（无需提前选中）
- 拖动片段靠近另一片段 ≤8px 时自动吸附对齐
- 缩放轨道右键菜单可添加缩放块

## 2. 功能需求

### 2.1 F1：编辑器快捷键

#### F1.1 Space — 播放/暂停

- **范围**：整个编辑器（timeline + preview），非焦点输入框/弹窗内
- **行为**：
  - 播放中按 Space → 暂停，播放头停在当前位置
  - 暂停/停止中按 Space → 从当前播放头位置开始播放
- **不改变**：工具栏播放按钮的行为和状态同步

#### F1.2 X — 切割视频片段

- **范围**：Timeline widget 获得焦点时
- **行为**：
  - 检测播放头下方是否有视频轨道的 Clip 覆盖
  - 有 → 在播放头位置执行 `SplitClipCommand`，切割该 Clip 为两段
  - 无 → 不执行任何操作，状态栏提示"播放头下无视频片段"
- **约束**：
  - 切割操作可撤销/重做
  - 仅作用于视频轨道（`type="video"`），不切割音频轨道
  - 播放头恰好在 Clip 边缘时不执行（避免产生零宽片段）
- **与 S 键的区别**：S 键切割当前选中片段（鼠标点击选中），X 键自动定位播放头下的视频片段

### 2.2 F2：视频片段吸附

- **范围**：同一视频轨道内的片段
- **吸附阈值**：8px（逻辑像素）
- **行为**：
  - 拖动片段靠近另一片段时，当两者边缘距离 ≤8px 自动对齐到 0 间隙
  - 吸附点：拖动片段的左/右边缘与目标片段的右/左边缘
  - 吸附发生时显示临时对齐线（垂直虚线，颜色与时间线主题一致）
- **不吸附**：不同轨道片段、片段与播放头、片段与时间刻度、音频轨道片段
- **约束**：
  - 移动操作可撤销/重做（复用现有 `MoveClipCommand`，吸附发生在拖动过程中）
  - 不改变 Clip 的 `start`/`end` 之外的属性

### 2.3 F3：手动添加缩放块

- **范围**：缩放轨道（`type="zoom"` 的 Track）
- **入口**：
  - 在缩放轨道非片段区域右键 → 弹出菜单 →"添加缩放块"
  - 保留现有双击缩放轨道空白区域添加缩放块的行为
- **创建位置**：点击所在的时间轴位置（记录为 `Clip.start`）
- **默认参数**：
  - 持续时间：2 秒
  - 矩形：屏幕中央，保持与项目画面一致的宽高比
  - `transition_duration`：0.4 秒（与现有默认值一致）
- **创建后**：
  - 自动选中该缩放块
  - 预览区域显示 `ZoomOverlay`，可拖动/缩放矩形
- **约束**：
  - 缩放块支持移动、裁剪（边缘拖拽）、删除
  - 所有操作支持撤销/重做
  - 不引入缩放关键帧、缩放曲线编辑、缩放倍数输入

## 3. 用户故事

### US-1: 键盘播放控制和快速切割
**作为** 视频编辑用户
**我想要** 用 Space 键控制预览播放/暂停，用 X 键快速切割播放头下的片段
**以便** 双手不离开键盘即可高效编辑

**验收标准：**
- [ ] 在编辑器内按 Space，播放状态切换正确（播放↔暂停）
- [ ] Space 快捷键在输入框/弹窗获得焦点时不触发
- [ ] 播放头在视频片段上方时按 X，该片段在播放头处被切割为两段
- [ ] 播放头下无视频片段时按 X，不执行切割并显示提示
- [ ] X 切割结果与 S 键切割一致（source_start/source_end 正确计算）
- [ ] X 切割操作可撤销/重做

### US-2: 视频片段自动吸附
**作为** 视频编辑用户
**我想要** 拖动一段视频靠近另一段时自动吸附对齐
**以便** 无需手工微调即可消除间隙或重叠

**验收标准：**
- [ ] 同一视频轨道两片段边缘距离 ≤8px 时自动对齐
- [ ] 吸附发生时显示临时对齐线
- [ ] 吸附操作可撤销/重做
- [ ] 不同轨道片段之间不吸附
- [ ] 边缘距离 >8px 时不触发吸附

### US-3: 手动添加缩放块
**作为** 视频编辑用户
**我想要** 在缩放轨道右键添加缩放块
**以便** 直观地在指定位置创建缩放效果

**验收标准：**
- [ ] 缩放轨道空白区域右键出现"添加缩放块"菜单项
- [ ] 点击后在指定位置创建 2 秒默认缩放块
- [ ] 创建后缩放块自动选中且显示 ZoomOverlay
- [ ] 创建操作可撤销/重做

## 4. Out of Scope

| 排除项 | 原因 |
|--------|------|
| JKL 播放控制 | 属于播放定位范畴，需独立设计交互 |
| 多选批量切割 | 当前为单选模型，多选需改造选择系统 |
| 跨轨道片段拖动 | `MoveClipCommand` 支持但 UI 无交互，应单独立项 |
| 播放头吸附 | 与片段吸附交互冲突，需独立评估 |
| 缩放关键帧模型 | 当前为基于片段的分段常量模型，重构影响面大 |
| 缩放块工具栏按钮 | 右键菜单已满足入口需求，工具栏按钮可后续加 |
| 缩放速度/曲线编辑 UI | 超出本次范围 |
| 音频轨道切割（X 键） | X 键仅作用于视频轨道，音频切割继续使用现有交互 |

## 5. 技术约束

- 语言：Python 3.11+
- UI 框架：PyQt5
- 切割命令必须继承 `UndoCommand`，纳入现有 undo/redo 栈
- 吸附逻辑在 `mouseMoveEvent` 拖动过程中计算，不影响 `MoveClipCommand` 的 undo 数据
- 快捷键在 `TimelineWidget.keyPressEvent` 中处理，需跳焦点检查
