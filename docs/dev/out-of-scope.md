# Out-of-Scope

## Recordly 核心稳定性与架构治理（2026-07-16）

以下内容经需求访谈明确排除，不属于本轮完成定义：

| 排除项 | 原因 |
|--------|------|
| 历史项目自动迁移 | 本轮只保证不兼容项目失败时明确提示且不破坏当前会话；不承担旧 schema 转换成本 |
| macOS 系统音频录制 | 当前完整验收平台为 Linux，Windows仅保持兼容；macOS 需要独立采集方案 |
| FPS 转换或帧重采样 | 本轮导出保持项目原始 FPS，避免引入重复帧、丢帧和额外同步复杂度 |
| 麦克风和系统音轨独立调音 UI | 本轮持久化独立音轨，但继续按现有规则混音 |
| 日志文件与日志轮转 | 使用 `logging` + stderr 已满足本轮诊断需求 |
| 远程错误上报 | 涉及隐私、服务端和合规设计，应单独立项 |
| 新编辑功能、云同步、账号系统 | 与本轮稳定性治理无关 |
| Release、安装包发布、应用商店分发 | 本轮终点为代码合并和验收，不执行发布 |

重新纳入任何排除项前，必须更新 PRD、技术方案和任务 DAG。

## 时间线裁剪功能完善（2026-07-17）

以下内容经需求访谈明确排除：

| 排除项 | 原因 |
|--------|------|
| 视觉拖拽手柄（E1） | 先用光标提示检验可用性，后续迭代再加 |
| Ripple Delete（E2） | 增加复杂度，先保证基础裁剪正确 |
| 多轨裁剪联动 | 当前仅视频+音频双轨，无多轨场景 |
| 裁剪预览（拖拽时实时更新画面） | 拖拽时预览帧刷新已有性能问题，需单独优化 |
| JKL 精确定位 | 属于播放控制范畴，非裁剪专属 |

重新纳入任何排除项前，必须更新 PRD、技术方案和任务 DAG。
